# 安全模式工作台 — 新 UI 设计说明

本文档描述 CodexStudy 桌面端 **安全模式（codex-new）工作台** 的目标形态、与主页的分工、布局结构、交互规则及实现方向。供产品与开发对齐，细节可在实现前继续补充。

---

## 1. 背景与目标

### 1.1 主页 vs 工作台

| 场景 | 界面 | 职责 |
|------|------|------|
| 普通开发 | **主页** | 对话、写代码、Git、Agent 改动预览等日常流程 |
| 启用安全模式后 | **工作台**（独立 Tauri 窗口） | 可视化「原项目 ↔ 隔离克隆」的差异，按对话执行合并、回滚、回溯、审查等操作，并查看 AI 在沙盒中的终端执行 |

工作台 **不是** 第二个完整 IDE，而是安全模式下的 **运维 / 对照可视化服务**。

### 1.2 设计原则

- **左栏**：双目录树，与磁盘状态同步，便于观察操作对两侧目录的影响。
- **中栏**：竖向会话列表，可展开；**同一套 UI**，每个会话 **状态不同**（非多套独立页面）。
- **代码**：点击文件后查看；明确区分 **新建 / 修改 / 删除**；修改采用 Cursor 式 diff 高亮；支持 **按 hunk 部分回滚**。
- **右栏**：当前会话的 AI 终端沙盒（命令、输出、运行状态）。
- **右侧 Git 面板（主页）**：仅保留安全模式开关、简短状态、「打开工作台」；**不**占用数据目录树、CODEX_HOME 等诊断信息（这些在工作台或设置中按需查看）。

## 2. 整体布局

```
┌─────────────┬──────────────────────────────┬──────────────────┐
│ 左：双目录树  │ 中：会话工作台（竖向列表）        │ 右：终端沙盒       │
│             │                              │                  │
│ [原项目]     │  ▼ 对话 A  ← 展开/收起          │  命令 + 输出流    │
│  文件树…     │     · 变更摘要 / 文件列表         │  （绑定当前会话）  │
│             │     · 合并 / 回滚 / 回溯 / 审查   │                  │
│ [隔离克隆]   │  ▶ 对话 B                      │                  │
│  文件树…     │  ▶ 对话 C                      │                  │
│             │                              │                  │
│ 点击文件 ─────┼──→ 代码 / diff 视图（见 §4）     │                  │
└─────────────┴──────────────────────────────┴──────────────────┘
```

建议窗口标签仍为 `codex-new-process`（进程 + 终端已合并为单一「工作台」入口）。

## 3. 各区域说明

### 3.1 左栏 — 双目录树

**两棵树，两个根：**

1. **原项目**（`originalRoot`）— 用户真实工程目录。
2. **隔离克隆**（`workspaceRoot`）— AI / 测试实际执行的目录。

**「与磁盘同步」的含义：**

- 不是把克隆实时 mirror 成原项目。
- 而是在合并、回滚、回溯、AI 改文件、`refresh_changes`、切换会话等事件后，从磁盘 / manifest **刷新** 两棵树的展示与状态角标。
- **刷新策略**：防抖 300ms，避免频繁操作时的性能问题。

**文件状态角标（可访问性优化）：**

| 状态 | 图标 | 颜色 | 含义 | 典型来源 |
|------|------|------|------|----------|
| 新增 | ➕ | 绿色 | 克隆中有，原项目中无 | `changedFiles.status === "added"` |
| 修改 | ✏️ | 橙色 | 两侧路径存在，内容不同 | `status === "modified"` |
| 删除 | ❌ | 红色 | 克隆侧标记删除等 | `status === "deleted"` |
| 已合并 | ✓ | 蓝色 | 已写入原项目 | `accepted === true` / merge 记录 |
| 冲突 | ⚠️ | 黄色 | 合并失败或 hash 不一致（见 §9.2 分期） | merge conflict / API 错误 |

**搜索与过滤**：

- 顶部搜索框：支持文件名模糊搜索（快捷键 `Ctrl+F`）
- 过滤按钮：
  - 「仅显示变更」（默认关闭，显示完整树）
  - 「仅显示待合并」
  - 「仅显示冲突」

**性能优化**：

- 文件数 < 1000：直接渲染
- 文件数 1000-5000：虚拟滚动（复用主页 `@tanstack/react-virtual`，与 `FileTreePanel` 一致）
- 文件数 > 5000：懒加载（目录默认折叠）

点击左树文件 → 打开 §4 的代码 / diff 视图（与中间列表点击同一数据源）。

### 3.2 中栏 — 竖向会话工作台

**布局：竖向列表，非顶部横向 Tab。**

每个 **对话（thread）** 对应一条可展开项；展开后显示该会话专属 **状态与操作**（UI 组件复用，数据按 `threadId` / `taskId` 隔离）。

**收起时（摘要）：**

- 对话标题 / 工作区名
- 待合并文件数、运行中终端等简要指标

**展开时（详情）：**

```
▼ 对话 A · pvz · 2 个待合并
    ├ 变更：3 新增 · 5 修改 · 1 删除
    ├ [文件列表] → 点击打开 diff
    ├ 合并到原项目（可选 hunk）
    ├ 回滚已合并（可选 hunk）
    ├ 回溯（project / workspace 目标）
    └ 审查 / 测试状态（若项目策略要求）
```

**会话切换规则：**

- 同一时间 **仅一个会话处于「当前激活」**（深色高亮 + 展开详情 + 右栏终端绑定）
- 其他会话可同时展开查看摘要（浅色显示），但不绑定终端和 diff 视图
- 切换会话时：
  - 自动保存当前会话的 hunk 勾选状态（存储在 `sessionStorage`）
  - 加载目标会话的 task state 和已保存的勾选状态
  - 右栏终端切换到目标会话的命令历史

**快捷键支持**：

- `Ctrl+↑/↓`：切换激活会话
- `Space`：展开/收起当前会话
- `Enter`：激活当前选中会话

**与后端模型对应：**

- `threadRegistry[threadId]` → 隔离目录、标题等
- `activeTask`（按会话 focus 后）→ `changedFiles`、`diff`、`review`、merge 状态
- traceback → **按 task 目录** 存储，带 `revision`，各会话历史独立

### 3.3 右栏 — 终端沙盒可视化

- 展示 **当前选中会话** 下 AI / 测试执行的命令与输出（`terminalRuns`、command runs）。
- 工作目录为隔离克隆（`workspaceRoot`）。
- 会话切换时过滤或切换绑定，不与其它会话混显。

底部可保留可折叠终端 dock（与现有 `CodexNewTerminalDock` 概念一致），作为右栏或底栏实现细节。

## 4. 代码与 Diff 查看

### 4.1 入口与布局

- **默认不** 占满屏编辑器；用户 **点击文件**（左树或中间列表）后打开代码 / diff 区域。
- **区域位置**：嵌在中栏展开区下方（非独立第四列），保持三栏布局简洁。
- **关闭方式**：点击 diff 区域右上角的 `×` 或按 `Esc` 键。

### 4.2 新建 vs 修改 vs 删除

| 类型 | 展示方式 | 图标 |
|------|----------|------|
| **新建** | 仅展示克隆侧全文；标题标明「新建文件」；无原项目侧内容 | ➕ |
| **修改** | Cursor 式 diff：增删行高亮（复用主页 `GitDiffViewer` / `ds-diff` 能力）；对比 **原项目 vs 隔离克隆**（默认） | ✏️ |
| **删除** | 标明删除语义；展示原项目侧将被影响的内容或说明 | ❌ |

### 4.3 Diff 对比模式切换

**默认模式**：原项目 vs 隔离克隆（查看 AI 改了什么）

**切换按钮**：
```
┌─────────────────────────────────────────┐
│ src/main.rs                             │
│ [原项目 vs 克隆 ▼] [查看历史]            │  ← 下拉菜单
├─────────────────────────────────────────┤
│ - old line                              │
│ + new line                              │
└─────────────────────────────────────────┘
```

**可选模式**：
- 原项目 vs 克隆（默认）
- 克隆改前 vs 克隆改后（回溯场景）
- 三路合并视图（**Phase 4+**，冲突解决时，见 §9.2）

### 4.4 部分回滚（按 hunk）

- 在 diff 视图中 **按 hunk 勾选**，再执行合并或回滚。
- 与现有 `selectedMergeHunks` / `selectedRollbackHunks` 及后端 hunk 级 API 对齐，**不**另造全局 Undo 栈。
- **快捷键**：
  - `Ctrl+A`：全选 hunk
  - `Ctrl+D`：取消全选
  - `Space`：切换当前 hunk 勾选状态

### 4.5 回溯 vs 回滚（UI 语义分离）

| 操作 | 主要影响 | UI 建议 | 快捷键 |
|------|----------|---------|--------|
| **合并** | 原项目文件 | 「合并到原项目」 | `Ctrl+M` |
| **回滚已合并** | 撤销已写入原项目的合并（**不是**编辑器 Undo） | 「回滚已合并」 | `Ctrl+Shift+Z`（**勿用** `Ctrl+Z`，避免与系统撤销混淆） |
| **回溯 restore** | 按 traceback 恢复 project 或 workspace 侧单文件快照 | 「回溯」+ 目标选择 | `Ctrl+Shift+R` |

三者不可混为一个「撤销」按钮；各会话 traceback **revision 顺序独立**。快捷键可在实现前统一校对，但须避免占用 `Ctrl+Z`。

## 5. 操作与左树刷新的关系

| 操作 | 原项目树 | 克隆树 | 刷新策略 |
|------|----------|--------|----------|
| AI 编辑 | 通常不变 | 变更 | 防抖 300ms |
| 合并到原项目 | 变更 | manifest 状态更新 | 立即刷新 |
| 回滚已合并 | 恢复 | 视策略 | 立即刷新 |
| 回溯 (project) | 指定文件恢复 | 不变或联动 refresh | 立即刷新 |
| 回溯 (workspace) | 不变 | 指定文件恢复 | 立即刷新 |
| 切换会话 | 不变 | 加载新会话状态 | 立即刷新 |

**刷新机制**：

- 每次上述操作成功后触发 **refresh**（`refreshCodexNewChanges` / 后端 `refresh_changes`）
- AI 编辑等高频操作使用 **防抖 300ms**，避免频繁刷新
- 用户主动操作（合并、回滚）**立即刷新**，保证左树与中栏列表一致
- 刷新失败时显示错误提示，提供「重试」按钮

## 6. 与现有实现的关系

### 6.1 应复用

- `useCodexNewState`、Tauri 命令（merge / rollback / traceback / review / test / preview）
- 主页 diff 组件（`GitDiffViewer` 或抽取的双路径 diff 适配层）
- FileTree 构建与虚拟滚动逻辑（`FileTreePanel` 思路）
- Hunk 级选择与现有 Merge / Rollback 面板逻辑
- 终端 run 数据与 `CodexNewTerminalDock`

### 6.2 不建议继续堆叠

- 当前 `CodexNewProcessWindow.tsx` 单文件多 Tab + 顶部重复路径块（`CodexNewDataPathsPanel` + 双份 `DirectoryHierarchy`）
- 会话侧栏 `scrollIntoView` 导致整页滚动跳动（已识别问题）

### 6.3 推荐架构

在同一 Tauri 窗口（`codex-new-process`）内新增 **`WorkbenchShell`**（名称可调整），拆模块：

| 模块 | 职责 |
|------|------|
| `DualTreePanel` | 左栏双根目录树 + 状态角标 + refresh |
| `SessionWorkbench` | 中栏竖向会话列表 + 展开详情 + 操作区 |
| `FileDiffPane` | 点击文件后的 diff / 预览 + hunk 选择 |
| `SandboxTerminal` | 右栏终端 |
| 现有 Merge / Rollback / Traceback / Review 面板 | 作为 SessionWorkbench 子模块迁入，非首屏 |

**不需要** 新开第二个 Tauri 窗口类型；**需要** 新布局壳而非在旧 Process 页上继续 patch。

## 7. 主页右侧 Git 面板（安全模式卡片）

开启安全模式后卡片仅展示：

- 标题、Active/Inactive 徽章、简短说明
- 工作区名、对话名（两行 meta）
- **一个「工作台」按钮**（原「进程 + 终端」已合并）

**不展示：**

- CODEX_HOME / 安全模式数据根 / 目录布局树
- 本地目录 `codex-new/workspaces/...` 长路径块

数据路径类信息可在工作台内「⋯ → 数据目录」或设置中按需打开。

## 8. 关键决策（已优化）

### 8.1 Diff 默认对比对 ✅

**决策**：默认 **原项目 vs 隔离克隆**，但提供切换按钮。

**理由**：
- 主要场景是"查看 AI 改了什么，是否合并到原项目"
- 回溯场景需要看克隆内历史变化时，提供「查看历史」按钮切换到"克隆改前 vs 改后"模式

**UI 实现**：
```
┌─────────────────────────────────────────┐
│ src/main.rs                             │
│ [原项目 vs 克隆 ▼] [查看历史]            │  ← 切换按钮
├─────────────────────────────────────────┤
│ - old line                              │
│ + new line                              │
└─────────────────────────────────────────┘
```

---

### 8.2 代码面板位置 ✅

**决策**：中栏展开区下方嵌入（非独立第四列）。

**理由**：
- 保持三栏布局简洁
- 文件与 diff 在同一视觉流中，减少视线跳转
- 独立第四列会让窗口过宽，且难以适配小屏幕

**布局示例**：
```
┌─────────────┬──────────────────────────────┬──────────────────┐
│ 左：双目录树  │ 中：会话工作台                 │ 右：终端沙盒       │
│             │  ▼ 对话 A                     │                  │
│             │     · 文件列表                 │                  │
│             │     · 操作按钮                 │                  │
│             │  ┌─────────────────────────┐ │                  │
│             │  │ Diff 视图（点击文件后展开）│ │                  │
│             │  │ - old line              │ │                  │
│             │  │ + new line              │ │                  │
│             │  └─────────────────────────┘ │                  │
└─────────────┴──────────────────────────────┴──────────────────┘
```

---

### 8.3 多会话展开策略 ✅

**决策**：仅一个会话「当前激活」（展开 + 右栏终端绑定），但允许多个会话同时展开查看摘要。

**规则**：
- **激活会话**（深色高亮）：展开详情 + diff + 右栏终端绑定
- **非激活会话**（浅色）：可展开查看文件列表和摘要，但不绑定终端
- 点击非激活会话的「激活」按钮 → 切换激活状态

**状态保存**：
- 未提交的 hunk 勾选 **按 `threadId` 分 key** 存 `sessionStorage`，例如 `codex-new:hunks:${threadId}`
- 扩展现有 `useCodexNewState`，**不**为工作台单独引入 Zustand
- 切换会话时：先写当前 key，再读目标 key

---

### 8.4 完整目录树策略 ✅

**决策**：左栏默认显示 **完整目录树**，但超过阈值时启用虚拟滚动和懒加载。

**性能阈值**：
- 文件数 < 1000：直接渲染完整树
- 文件数 1000-5000：虚拟滚动（仅渲染可见区域）
- 文件数 > 5000：懒加载（目录默认折叠，点击时加载子节点）

**搜索/过滤**：
- 顶部提供搜索框，支持文件名模糊搜索
- 过滤选项：「仅显示变更文件」「仅显示待合并」「仅显示冲突」

**实现参考**：
- 直接复用 / 抽取 `FileTreePanel` 的树构建与 `@tanstack/react-virtual`，不新增第二套虚拟列表库

---

### 8.5 只读 vs 可编辑 ✅

**决策**：首版 **只读预览 + diff + hunk 操作**，不支持在工作台内直接编辑。

**理由**：
- 避免引入"在工作台编辑 → 状态同步 → 与 AI 改动冲突"的复杂度
- 工作台定位是"审查和合并"，不是"编辑器"
- 如需修改，用户应在主页编辑器中操作

**未来扩展**（Phase 4+）：
- 如果用户强烈需求，可考虑"快速修正"功能（仅限单行小改）
- 但需要明确的冲突解决策略和 undo 机制

---

### 8.6 与主页 Agent edits 的一致性 ✅

**决策**：以 **工作台 + 后端 manifest** 为准，主页仅保留快速预览。

**分工**：
- **主页**：实时显示 AI 正在改动的文件（轻量预览，无 hunk 操作）
- **工作台**：完整的 diff、hunk 选择、合并、回滚、回溯

**数据流**：
```
AI 编辑 → 后端 manifest 更新 → 触发事件
                ↓
        ┌───────┴───────┐
        ↓               ↓
    主页预览        工作台刷新
   (只读卡片)      (完整操作)
```

---

### 8.7 实现约束与澄清（避免踩坑）

| 点 | 约定 |
|----|------|
| **快捷键** | 回滚已合并不用 `Ctrl+Z`；与 §4.5 一致 |
| **冲突 UI** | **MVP（Phase 2–3）**：合并失败 Toast + 冲突文件列表 + 阻断，**不**做完整三路编辑器 |
| **冲突 UI 进阶** | **Phase 4+**：三路合并视图（需核对 `codex-new-core` 是否暴露 per-file conflict 状态） |
| **错误范围** | 工作台以 **本地磁盘 / Tauri invoke / manifest** 为主；远程 Git 失败归 **主页 Git 面板**，不在此重复一套 |
| **虚拟滚动** | 仅 `@tanstack/react-virtual`（与 `FileTreePanel` 一致） |
| **状态** | hunk 勾选按 `threadId` 存 `sessionStorage`；业务状态仍走 `useCodexNewState` |
| **排期** | **MVP ≈ Phase 1–2**（三栏 + 会话 + diff + hunk + 终端）；Phase 4 冲突三路、WCAG 全量可标 P1 裁剪 |

## 9. 错误处理与用户反馈

### 9.1 错误分类与处理

| 错误类型 | 场景示例 | UI 反馈 | 恢复策略 |
|---------|---------|---------|---------|
| **合并冲突** | 原项目文件已被手动修改 / hash 不一致 | Toast + 冲突文件列表；左树 ⚠️ 角标 | **MVP**：取消合并、逐文件重试；**进阶**：三路合并（§9.2） |
| **回溯失败** | 目标 revision 不存在 | Toast："该版本快照不存在" | 列出可用 revision 供选择 |
| **刷新失败** | 磁盘 I/O、`refresh_changes` 失败 | 顶部横幅："无法读取文件变更" | 「重试」「查看日志」 |
| **权限错误** | 无法写入原项目目录 | 模态框说明路径与权限 | 提示检查目录权限或以管理员重试（平台相关） |
| **调用失败** | Tauri 命令超时 / 后端 panic | Toast："操作失败" + 错误摘要 | 重试；开发模式可展开详情 |

### 9.2 冲突解决流程

**触发条件**：合并时检测到原项目文件已被修改（或后端返回 merge conflict）。

**分期**：
- **MVP**：下列列表 + 「取消合并」；**不做**内嵌三路编辑器。
- **进阶（Phase 4+）**：三路合并视图（实现前确认后端字段与 API）。

**UI 展示（MVP 起）**：
```
┌─────────────────────────────────────────┐
│ ⚠️ 检测到 3 个文件冲突                    │
│ [查看冲突] [取消合并] [强制覆盖]          │
└─────────────────────────────────────────┘

冲突文件列表：
  ⚠️ src/main.rs
  ⚠️ src/utils.rs
  ⚠️ package.json
```

**三路合并视图**：
```
┌──────────────┬──────────────┬──────────────┐
│ 原项目（基础） │ 你的修改      │ AI 的修改     │
│ old content  │ your change  │ ai change    │
└──────────────┴──────────────┴──────────────┘
         [采用左侧] [采用右侧] [手动编辑]
```

### 9.3 操作确认

**需要确认的操作**（显示确认对话框）：
- 回滚已合并（影响原项目）
- 强制覆盖（忽略冲突）
- 删除会话（清理隔离目录）
- 批量合并（超过 10 个文件）

## 10. 数据持久化与状态恢复

### 10.1 需要持久化的状态

| 状态 | 存储位置 | 生命周期 | 用途 |
|------|---------|---------|------|
| **Hunk 勾选** | `sessionStorage`（key：`codex-new:hunks:${threadId}`） | 浏览器会话 / 关窗可清 | 切换 thread 时恢复该对话的勾选 |
| **会话展开状态** | `localStorage` | 永久 | 重新打开工作台时恢复展开状态 |
| **左树折叠状态** | `localStorage` | 永久 | 记住用户的目录折叠偏好 |
| **Diff 对比模式** | `localStorage` | 永久 | 记住用户偏好的对比模式 |
| **窗口布局** | `localStorage` | 永久 | 左/中/右栏宽度比例 |
| **搜索历史** | `localStorage` | 永久（最多 20 条） | 文件搜索的历史记录 |

### 10.2 状态恢复流程

**工作台启动时**：
1. 从后端加载 `threadRegistry` 和 `changedFiles`
2. 从 `localStorage` 恢复会话展开状态和窗口布局
3. 从 `sessionStorage` 恢复上次激活会话的 hunk 勾选
4. 如果上次激活会话已不存在，激活第一个会话

**会话切换时**：
1. 保存当前会话的 hunk 勾选到 `sessionStorage`
2. 加载目标会话的 hunk 勾选（如果存在）
3. 刷新左树和 diff 视图

**窗口关闭时**：
1. 保存当前会话展开状态到 `localStorage`
2. 保存窗口布局到 `localStorage`
3. 清理过期的 `sessionStorage` 数据（超过 7 天）

### 10.3 数据清理策略

- **自动清理**：已完成且超过 30 天的会话数据
- **手动清理**：工作台设置中提供「清理历史数据」按钮
- **磁盘空间警告**：隔离目录超过 5GB 时显示警告

## 12. 性能优化

### 12.1 渲染优化

**虚拟滚动**：
- 左树文件数 > 1000 时启用
- 使用 `@tanstack/react-virtual`（与 `FileTreePanel` 相同，不引入 `react-window`）
- 仅渲染可见区域 ±10 项

**懒加载**：
- 目录默认折叠，点击时加载子节点
- Diff 视图按需加载（点击文件时才请求）
- 终端输出分页加载（每次 1000 行）

**防抖与节流**：
- 搜索输入：防抖 300ms
- 窗口 resize：节流 100ms
- 文件变更刷新：防抖 300ms

### 12.2 数据缓存

**前端缓存**：
- 文件树结构缓存 5 分钟
- Diff 内容缓存（LRU，最多 50 个文件）
- 会话列表缓存 1 分钟

**后端优化**：
- `refresh_changes` 仅返回变更的文件
- 增量 diff 计算（仅计算请求的文件）
- 优先 manifest / `refresh_changes` 增量结果；若后端已用 git 统计可保留，**不以 Git 网络操作为工作台前提**

### 12.3 性能监控

**关键指标**：
- 工作台启动时间 < 2s
- 会话切换延迟 < 500ms
- Diff 视图加载 < 1s
- 左树刷新 < 300ms

**性能日志**：
- 记录慢操作（> 1s）到控制台
- 提供性能分析面板（开发模式）
